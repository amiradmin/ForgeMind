# ForgeMind Architecture Rules

## 1. Modular Monolith
We start with a modular monolith. Microservices only when necessary.

## 2. Domain Driven Design
Core domains:
- Asset
- Maintenance
- Energy
- Inventory
- AI
- Computer Vision

## 3. Clean Architecture
No business logic inside views.

## 4. Testability
Every module must be independently testable.

## 5. Documentation First
No feature without documentation.
